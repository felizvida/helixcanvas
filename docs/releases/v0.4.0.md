# HelixCanvas v0.4.0

HelixCanvas `v0.4.0` is the release where the **personal biomedical figure workstation** direction becomes much more tangible.

## Highlights

- desktop-grade macOS packaging with a branded app icon and native project-file open/save dialogs
- safer local-first behavior through hardened persistence, base-path-safe offline support, and more truthful AI edit reporting
- faster blank-canvas escape through starter templates for oncology, immunology, microscopy, neuroscience, infection, and neutral workflows
- deeper scientific scaffolds with builder variants and style presets for pathways, trafficking maps, spatial profiling, dose response, and recovery timelines

## Included In This Release

- Electron desktop shell and native file bridge in `desktop/main.cjs` and `desktop/preload.cjs`
- embedded local server startup refactor in `server/index.mjs`
- desktop icon generation in `scripts/build-desktop-icon.sh`
- persistence and workspace storage helpers in `src/lib/persistence.js` and `src/lib/workspaceStore.js`
- app-base-safe runtime helpers in `src/lib/appPaths.js`
- starter-template data in `src/data/starterTemplates.js`
- richer scientific builder system in `src/lib/scientificBuilders.js`
- expanded domain presets and workbench UI in `src/lib/domainPresets.js` and `src/App.jsx`

## Validation

The release was validated with:

```bash
npm run check
npm run desktop:dir
```

## Release Intent

This cut is meant to make HelixCanvas feel more like a trustworthy daily driver for one researcher working locally:

- native-feeling project files instead of browser-only save flows
- safer persistence when projects, imports, and snapshots get larger
- better first drafts through guided starter templates and configurable scientific scaffolds
- a more polished desktop identity through packaged app output instead of only the browser shell
