# HelixCanvas v0.4.2

HelixCanvas `v0.4.2` is an editor-depth release focused on making the canvas feel more like a mature biomedical figure workstation.

## Highlights

- multi-object transform handles with resize, rotate, flip, match-size, and fit-to-board workflows
- graph-aware connector transforms when selected diagrams move, resize, rotate, or flip
- smart connector creation from selected layers, endpoint anchoring, curved routes, dashed/dotted styles, and on-canvas curve bending
- asset crop, fit, mask, zoom, opacity, flip, and visual-effect controls with SVG export fidelity
- fragment copy/cut/paste that preserves nodes, internal connectors, remapped anchors, groups, and linked review comments
- style copy/paste for layers and connectors
- top-first layer order panel, per-layer up/down controls, one-step order commands, and front/back jumps
- batch layout intelligence for tidy rows, columns, grids, and radial rings

## Included In This Release

- editor transform and layout helpers in `src/lib/editorSelection.js`
- connector geometry, anchoring, curve, dash, and smart-draft helpers in `src/lib/connectors.js`
- graph-safe fragment clipboard helpers in `src/lib/sceneClipboard.js`
- layer and connector style transfer helpers in `src/lib/styleClipboard.js`
- export fidelity improvements in `src/lib/exporters.js`
- expanded editor UI and keyboard shortcuts in `src/App.jsx`
- new editor guide in `docs/EDITOR_GUIDE.md`
- updated roadmap, product, personal-workbench, tutorial, and GitHub milestone documentation

## Validation

The editor-depth work was validated with:

```bash
npm run check
```

The current suite includes pack validation, unit tests, and production build.

## Release Intent

This cut narrows the practical gap between HelixCanvas and mature biomedical illustration editors for personal use:

- users can manipulate diagrams as structured scene graphs rather than loose objects
- pathway arrows can remain attached to biological objects
- repeated motifs can be copied without flattening provenance or editability
- common figure arrangements can be cleaned up quickly
- exports preserve more of the visual state users see on the board

The next editor-focused work should prioritize richer scientific text, connector labels and pathway annotations, and continued interface simplification.
