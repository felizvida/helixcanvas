# HelixCanvas v0.2.0

HelixCanvas `v0.2.0` is the first release that feels like an open-source product instead of a prototype.

## Highlights

- local-first editor workflows with project save/open, recovery drafts, lock/hide layer controls, and stronger export handling
- validated asset packs with built-in Bioicons, Servier originals, and a documented contribution path for community packs
- real-world example figures for signaling, CRISPR workflow, and retinal pathology
- a tutorial set with generated SVGs, project files, and citation bundles under `docs/tutorial/`
- stronger contributor scaffolding, CI checks, and pack validation

## Included In This Release

- source-aware library improvements and pack-level provenance UI
- example-project loading from the app sidebar and direct example URLs
- `npm run build:tutorial` for regenerating tutorial artifacts from shipped example projects
- updated README, product docs, contributing docs, and asset pack specification

## Validation

The release was validated with:

```bash
npm run build:tutorial
npm run check
```

## Release Intent

This cut is meant to be a stable public starting point for:

- researchers building publication figures from open assets
- educators who need reusable teaching schematics
- contributors extending HelixCanvas with packs, docs, and editor improvements
