# HelixCanvas v0.4.1

HelixCanvas `v0.4.1` is a polish release focused on making the open-source personal biomedical figure studio easier to learn, more visually confident, and more useful with optional AI-generated content.

## Highlights

- optional OpenAI Image 2 generation for provenance-tagged, user-generated bitmap assets
- refreshed visual direction with a bolder editorial workspace, darker canvas surround, and cleaner publication-board contrast
- a much deeper tutorial built around real biological problems and real interface screenshots
- regenerated tutorial artifacts with stable project metadata, direct launch URLs, learning goals, exercises, citations, and SVG exports
- tighter seeded example layouts so tutorial canvases render cleanly in the live editor and generated exports

## Included In This Release

- Image generation endpoint in `server/index.mjs` and `server/aiService.mjs`
- client image request helper in `src/lib/ai.js`
- Image 2 prompt panel, generated-asset placement, source filtering, and AI status updates in `src/App.jsx`
- generated-content source policy in `src/data/servier.js`
- refreshed application styling in `src/styles.css`
- expanded tutorial generator metadata in `scripts/generate-tutorial-assets.mjs`
- comprehensive tutorial rewrite in `docs/tutorial/README.md`
- refreshed tutorial screenshot and regenerated tutorial SVG/project artifacts in `docs/tutorial/`

## Validation

The release was validated with:

```bash
npm run build:tutorial
npm run check
```

## Release Intent

This cut makes HelixCanvas feel more teachable and more expressive for individual researchers:

- new users can follow real biological walkthroughs instead of reading a feature list
- AI-generated visual content is clearly separated from bundled open-library assets
- examples and screenshots better represent the intended visual quality of the editor
- the documentation now supports self-guided learning, workshops, and contributor onboarding
